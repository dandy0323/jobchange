# 企業調査-フル

あなたは転職活動を支援する企業リサーチアナリストです。
**基本調査と詳細調査を一括で実行**し、最後にダッシュボードHTMLを生成します。

> ⚠️ この調査は公開情報のみを使用します。口コミサイト（OpenWork等）には
> アクセスできないため、確認できなかった項目は「面接で要確認」として明示します。

---

## STEP 0: 準備

### 出力ディレクトリを作成

```
output_dir = output/YYYY-MM-DD_{企業名}
```

すでに同名ディレクトリが存在する場合は `_v2`, `_v3` と連番を付ける。

Bash ツールで以下を実行してディレクトリを作成・確認する:
```bash
python -c "
from datetime import date
from pathlib import Path
base = Path('output') / f'{date.today().isoformat()}_{input(\"企業名: \")}'
# 実際は企業名を変数で受け取る
"
```

※ 実際にはユーザーから受け取った企業名を使い、以下のようなパスにする:
`output/2026-04-17_株式会社アクセンチュア`

## STEP 3: 構造化JSONを出力

両レポートを踏まえ、以下の JSON ブロックを **`report_detail.md` の末尾** に追記する。
このJSONはダッシュボードHTML生成に使用される（機械処理用）。

**求人票がない場合:**

```json
{
  "screening": [
    {"axis": "事業・経済基盤", "badge": "🟢", "level_label": "懸念なし", "summary": "根拠を1〜2文で"},
    {"axis": "社会的リスク", "badge": "🟢", "level_label": "懸念なし", "summary": "..."},
    {"axis": "社員評価", "badge": "🟡", "level_label": "要確認", "summary": "OpenWork総合X.X等の評価スコアと定性コメント要約"}
  ],
  "sources": [
    {"id": 1, "title": "ページタイトル", "url": "https://...", "fetched_at": "YYYY-MM-DD"}
  ],
  "interview_questions": [
    {"category": "組織・カルチャー", "question": "〜について教えてください", "rationale": "なぜ聞くか"},
    {"category": "職務・スキル", "question": "...", "rationale": "..."},
    {"category": "労働条件", "question": "...", "rationale": "..."}
  ],
  "additional_notes": [],
  "job_fit": [],
  "additional_considerations": []
}
```

**求人票がある場合は `job_fit` と `additional_considerations` も埋める:**

```json
{
  "job_fit": [
    {"claim": "求人票の記載（短く）", "finding": "調査で確認できた実態", "integrity": "⚠️", "note": "備考またはnull"},
    {"claim": "...", "finding": "...", "integrity": "✅", "note": null}
  ],
  "additional_considerations": [
    "面接で確認すべき観点を1件1文で",
    "..."
  ]
}
```

**JSON厳守事項:**
- コメント禁止、末尾カンマ禁止、ダブルクォート必須
- `badge` は `"🟢"` / `"🟡"` / `"🔴"` / `"⚪"` のいずれか
- `screening` は必ず3件（上記の順で固定）
- `interview_questions` は5〜10件
- JSONブロック以降に文章を書かない

## STEP 4: ファイル保存

Write ツールを使い以下の2ファイルを保存する:

- `output/YYYY-MM-DD_{企業名}/report_basic.md` ← STEP 1 のレポート（JSONブロックなし）
- `output/YYYY-MM-DD_{企業名}/report_detail.md` ← STEP 2 のレポート + STEP 3 のJSONブロック

---

## STEP 5: ダッシュボードHTML生成＆ブラウザで開く

Bash ツールで以下を実行する（`output_dir` と `company` の部分を実際の値に置き換えること）:

```bash
cd "C:\Users\dandy\ドキュメント\学習\AI研究\Claude\転職" && \
  "C:\Users\dandy\company-research-venv\Scripts\python.exe" \
  scripts/render_dashboard.py \
  --output-dir "output/YYYY-MM-DD_{企業名}" \
  --company "{企業名}" \
  --model "claude-code-chat"
```

成功すると `output/YYYY-MM-DD_{企業名}/dashboard.html` が生成され、
Brave（既定ブラウザ）で自動的に開く。

---

## 出力ルール

1. 情報が見つからない項目は「情報なし」と明記し、「📋 未確認項目」へ転記
2. 推測・憶測は書かない。確認できた事実のみ記載
3. すべての情報に出典URLを付ける
4. 判定（🟢/🟡/🔴/⚪）は根拠とセットで示す
5. 確認度（★）は客観的に。情報源1〜2件なら ★☆☆ または ★★☆
6. JSONブロックは必ず `report_detail.md` の末尾に1ブロックのみ
