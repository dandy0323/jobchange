# 求人票整合性チェック（Phase 4 追加指示）

求人票が提供されている場合、**詳細調査レポートの末尾** に次の2セクションを必ず追加してください。

---

## 求人票との整合性

Markdown本文には、以下の形式の表を必ず含めること:

| 求人票の記載 | 調査で確認できた実態 | 整合性 | 備考 |
|---|---|---|---|
| 例: 平均残業20時間以内 | OpenWork: 残業平均35時間と報告 | ⚠️ | 部署差あり |
| 例: フルリモート可 | 採用ページでハイブリッド記載 | ❓ | 面接で要確認 |
| 例: 年休125日 | 就業規則との整合を確認 | ✅ | 一致 |

整合性凡例:
- ✅ **整合**: 求人票の記載と調査結果が一致
- ⚠️ **要確認**: 差異あり、または情報が食い違う
- ❓ **不明**: 公開情報で確認が困難

## 追加留意事項

求人票に **書かれていない** が、面接で必ず確認すべき点を箇条書きで列挙してください。
例: 評価制度の運用実態 / 配属ガチャの有無 / 試用期間の条件変更 等。

## JSON 側の追加フィールド（必須）

応答末尾の JSON ブロックに以下2フィールドを必ず含めてください:

```json
{
  "job_fit": [
    {
      "claim": "求人票の記載内容（短く）",
      "finding": "調査で確認できた実態",
      "integrity": "⚠️",
      "note": "備考（任意、無ければ null でなく省略可）"
    }
  ],
  "additional_considerations": [
    "面接で確認すべき追加事項を1件1文で",
    "..."
  ]
}
```

制約:
- `job_fit` は **3〜8件** 程度。重要度の高いものから並べる
- `integrity` は絵文字1つのみ: `"✅"` / `"⚠️"` / `"❓"`（`⚠️` は U+26A0 + U+FE0F）
- `additional_considerations` は 3〜6件、面接でそのまま読める質問形式でなくてよい
  （確認観点として整理する）
