# レポート出力の追加ルール（ダッシュボード連携）

既存の Skill（`企業調査-基本` / `企業調査-詳細`）で指定されている Markdown レポートを
**そのまま出力** した上で、**応答の末尾に必ず以下の構造化データ1ブロック**を
付け加えてください。このJSONはダッシュボードHTMLを生成する機械処理で使用されます。

## 出力形式（厳密な JSON）

応答の最後に、下記の形式の **1つだけ** の JSON コードブロックを必ず含めてください。
このブロックより後には**一切文章を書かない**でください。

```json
{
  "screening": [
    {
      "axis": "事業・経済基盤",
      "badge": "🟢",
      "level_label": "懸念なし",
      "summary": "判定根拠を1〜2文で"
    },
    {
      "axis": "社会的リスク",
      "badge": "🟢",
      "level_label": "懸念なし",
      "summary": "..."
    },
    {
      "axis": "社員評価",
      "badge": "🟡",
      "level_label": "要確認",
      "summary": "OpenWork総合X.X等の評価スコアと定性コメント要約"
    }
  ],
  "sources": [
    {
      "id": 1,
      "title": "ページタイトル",
      "url": "https://example.com/...",
      "fetched_at": "YYYY-MM-DD"
    }
  ],
  "interview_questions": [
    {
      "category": "事業戦略",
      "question": "面接で聞く質問文",
      "rationale": "なぜ聞きたいか（未確認項目との対応など）"
    }
  ],
  "additional_notes": []
}
```

## 厳守事項

1. **厳密な JSON**: コメント禁止、末尾カンマ禁止、ダブルクォート必須
2. **badge は絵文字1文字のみ**: `"🟢"` / `"🟡"` / `"🔴"` / `"⚪"` のいずれか
3. **screening は必ず3件**: 「事業・経済基盤」「社会的リスク」「社員評価」の順で必ず含める
   （情報が乏しい場合は `"badge": "⚪"` で "情報なし" と明記）
4. **sources の id**: 1から連番。Markdown本文中の `[1]` `[2]` … の番号と一致させる
5. **interview_questions**: Skill の「未確認項目」セクションから3〜8件程度、
   面接で聞ける形式（「〜について教えてください」等）に整えて収録
6. **末尾の位置**: 応答全体の最後に置き、その後に追加の文章・挨拶・サマリを**書かない**
7. **文字数上限**: 各 summary は 120 文字以内を目安。URLは短縮せずに完全な形で

## JSON が出せない場合

情報が極端に乏しく JSON を埋められないフィールドがある場合でも、**空配列 `[]` や
`"⚪"` `"情報なし"` を使って必ず構造を維持**してください。JSON ブロック自体の省略は不可です。
